📖 Project Overview

Smart Stadiums FIFA 2026

Complete Guide - Kya Hai, Kaise Chalta Hai

🎯 Project Kya Hai?

Smart Stadiums FIFA 2026 ek complete web application hai jo FIFA World Cup 2026 ke stadiums ke operations better karne ke liye banaya gaya.

4 Main Features

🤖 AI Stadium Companion

AI assistant jo multilingual support karta hai aur user questions ka jawab deta hai.

📊 Crowd Intelligence Dashboard

Real-time crowd data, occupancy tracking, aur 4-hour predictions.

🚨 Emergency Response System

Incident reporting, real-time tracking, aur AI recommendations.

🗺️ Smart Navigation

Indoor wayfinding, accessibility routing, aur crowd-aware path planning.

👥 Koun Istemaaal Karega?

User Type Kya Use Karega
Fans AI assistant, Navigation, Crowd Info
Organizers Dashboard, Crowd Analytics, Predictions
Security Staff Incidents, Emergency Response, Alerts
Volunteers Task Assignment, Communication
Admin System Management, Configuration

🏗️ Technology Stack

Frontend: React 19 + Next.js 15 + TypeScript + Tailwind CSS + Three.js Backend: Next.js API Routes + Prisma ORM + OpenAI GPT-4 Database: TiDB MySQL Deployment: Vercel

🏗️ Architecture Explained

Pura Flow Kaise Kaam Karta Hai?

1️⃣ User Signup/Login Flow

1. User aata hai: http://localhost:3000
↓
2. Login page dikhai deta hai
↓
3. Email + Password enter karta hai
↓
4. Browser → Server ko POST request: /api/auth/login
↓
5. Server → Database check karta hai
↓
6. Password match kare?
↓
7. Agar match → JWT token generate karta hai
↓
8. Token browser ko bhej deta hai
↓
9. Browser → Token localStorage mein save karta hai
↓
10. Dashboard page open hota hai ✓

2️⃣ AI Chat Flow

1. User kuch likhe: "Bathroom kidhar hai?"
↓
2. Browser → Server ko POST request: /api/ai/chat
↓
3. Server → Conversation history fetch karta hai
↓
4. Server → Context gather karta hai (location, role, previous messages)
↓
5. Server → OpenAI ko bhejta hai (context + message)
↓
6. OpenAI → Smart response generate karta hai
↓
7. Server → Response save karta hai database mein
↓
8. Browser → Chat interface mein message dikhai deta hai

3️⃣ Crowd Data Flow

Server har 30 seconds mein:
✓ Sensors/Mobile apps se crowd data collect karta hai
✓ Database mein store karta hai (crowd_analytics table)
✓ Predictions calculate karta hai
✓ Dashboard ko realtime update bhejta hai

🔌 API Endpoints

API Endpoints - Detailed

Authentication APIs

POST /api/auth/login

Request: { "email": "demo@stadiums.com", "password": "Demo@12345" } Response (Success): { "success": true, "data": { "user": { "id": "user_123", "email": "demo@stadiums.com", "fullName": "Demo User", "role": "fan" }, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." } }

POST /api/auth/register

Request: { "email": "newuser@example.com", "password": "Pass@12345", "fullName": "New User" } Response: Same as login (user + token)

AI Chat API

POST /api/ai/chat

Request: { "message": "Where's the bathroom?", "conversationId": null, "context": { "location": "Section A", "role": "fan", "language": "en" } } Response: { "success": true, "data": { "conversationId": "conv_123", "response": "The nearest restroom is at Section A, Level 2...", "tokens": 150 } }

Crowd Data API

GET /api/crowd/live-stats

Query: /api/crowd/live-stats?stadiumId=default-stadium Response: { "success": true, "data": { "totalOccupancy": 65.5, "zones": [...], "predictions": [...], "lastUpdated": "2026-07-07T15:32:00Z" } }

Incidents API

POST /api/incidents/report

Request: { "type": "medical", "severity": "high", "location": "Section A", "description": "Person fell down" } Response: { "success": true, "data": { "incidentId": "inc_123", "status": "reported", "createdAt": "2026-07-07T15:30:00Z" } }

🗄️ Database Explained

Database Schema - 14 Tables

Main Tables

1. users Table

Column Type Purpose
id String Unique identifier
email String Login ke liye
passwordHash String Encrypted password
fullName String User ka naam
role String fan/organizer/security/volunteer/admin

2. ai_conversations Table

Column Purpose
id Conversation ID
userId Kiska conversation hai
context Location, role, preferences
totalTokensUsed API call cost

3. crowd_analytics Table

Purpose: Real-time crowd data store karta hai (every 30 seconds)
• occupancyPct: 0-100 percentage • zoneData: JSON with zone details • timestamp: Record time

4. incidents Table

Purpose: Emergency reports store karta hai
• type: medical/security/fire • severity: low/medium/high/critical • location: Where incident happened • status: reported/acknowledged/resolved

Other Tables

seating_sections → Stadium ke sections seating_seats → Individual seats amenities → Bathrooms, food courts, etc gates → Entry/exit gates navigation_pois → Landmarks (Points of Interest) matches → Match events match_tickets → Ticket information user_notifications → Alerts aur notifications
